# ROI 대시보드 — 1단계 설계 문서

- **작성일:** 2026-07-08
- **작성자:** 류승환 (shryu@huno.kr) + Claude
- **범위:** 전체 제품의 1단계(기반 + 핵심 대시보드). 2·3단계는 별도 기획서로 진행.

---

## 1. 배경과 목표

현재 수익 관련 정보가 여러 개의 구글 시트·탭에 흩어져 있어, 고객사별·전사 수익률을 한눈에 파악하기 어렵다. 이를 대체하는 **자체 웹 대시보드**를 만든다.

**1단계 목표:** 수익 4종(계약금·실적·청구액·입금액)과 지출을 **수동 입력**으로 관리하고, 수익률을 한눈에 보여주는 완성형 내부 도구. 파일 자동 파싱과 구글 서비스 심화 연동은 이후 단계로 분리한다.

**성공 기준**
- @huno.kr 계정으로 로그인하면 역할에 맞는 대시보드가 뜬다.
- PM이 과업별 횟수를 입력하면 금액·합계·달성률·수익률이 자동 계산된다.
- 관리자/정산담당자가 지출·청구·입금을 입력하면 대시보드에 즉시 반영된다.
- PM은 본인 담당 고객사 데이터만 접근할 수 있다(DB 레벨에서 강제).

## 2. 전체 로드맵 (제품 분할)

| 단계 | 내용 | 상태 |
|---|---|---|
| **1단계** | 기반(인증·RBAC·데이터 모델) + 수동 CRUD + 수익률 대시보드 | **본 문서** |
| 2단계 | 업로드 자동화: 산출내역서 PDF→계약금·과업·단가, 세금계산서→청구액, 입금내역 엑셀→입금액 (양식 기반 파서 + AI 보조 + 사람 확인) | 이후 |
| 3단계 | 구글 심화 연동: 시트 이관·내보내기, 캘린더 연동 | 이후 |

각 단계는 자체 기획서 → 구현 계획 → 구현 주기를 갖는다.

## 3. 기술 아키텍처

- **프론트+백엔드:** Next.js (풀스택 React, App Router). 한 코드베이스로 화면·API 관리.
- **데이터베이스:** PostgreSQL. ORM은 Prisma.
- **배포:** Google Cloud — 앱은 Cloud Run, DB는 Cloud SQL(PostgreSQL).
  - **사유:** 재무 데이터가 구글 조직(GCP) 안에 있어야 한다는 요구. 계정·권한·데이터가 같은 구글 생태계에 머물러 보안·감사가 단순.
- **인증:** Auth.js(NextAuth) + Google OAuth.
  - @huno.kr 워크스페이스 도메인만 허용(hd 파라미터 + 서버 검증).
  - 별도 비밀번호 없음.
- **파일 저장(2단계 대비):** Google Cloud Storage. (1단계에선 미사용)

### 확장성
사용자 대규모·성장 대비 요구에 따라 처음부터:
- Cloud Run 오토스케일링, Cloud SQL 커넥션 풀링.
- 인덱싱: 조회의 축이 되는 (고객사, 연월), (PM), (역할) 기준 인덱스.
- 집계 쿼리는 월 단위 사전집계(materialized view 또는 집계 테이블)로 성능 확보 여지를 남긴다.

## 4. 권한 모델 (RBAC)

역할 위계: **관리자 > 정산담당자 > PM**

| 역할 | 조회 범위 | 주요 권한 |
|---|---|---|
| **관리자** | 전체 | 모든 데이터 조회·수정, 사용자·역할 관리, 고객사·과업·단가 설정, 시스템 설정 |
| **정산담당자** | 전체 | 청구·입금·지출 입력, 고객사·과업·단가 설정, 정산 전반. 사용자 관리 불가 |
| **PM** | **본인 담당 고객사만** | 담당 고객사의 실적 입력(횟수) 및 조회. 타 고객사·사용자관리·시스템설정 불가 |

- 역할은 **관리자가 앱 내에서** 사용자에게 지정한다(구글 로그인 후 첫 접근 시 관리자 승인/역할부여 흐름).
- **행단위 보안(Row-Level Security):** "PM은 본인 담당 고객사만"을 애플리케이션 레이어뿐 아니라 **PostgreSQL RLS 정책**으로도 강제해, 실수·버그로 인한 타 고객사 데이터 노출을 원천 차단.

## 5. 데이터 모델

핵심 엔티티와 관계 (1단계 기준):

```
User (사용자)
  - email (@huno.kr), name, role(ADMIN|SETTLEMENT|PM), status(승인대기|활성|비활성)

Client (고객사)
  - name, status(진행중|종료 등), 계약기간(start, end)
  - pm_id → User (담당 PM)

Task (과업)                        // 고객사에 속함
  - client_id → Client
  - name (심리진단·전문가상담·프로그램·홍보·운영관리 등, 자유 등록)
  - unit_price (단가)
  - contract_amount (계약금, nullable)   // 없을 수 있음
  - source (MANUAL | PDF)                // 2단계에서 PDF 자동생성 대비

MonthlyPerformance (월 실적)        // 과업 × 연월
  - task_id → Task
  - year, month
  - count (횟수, 입력값)
  - amount = unit_price × count (저장 또는 파생)

MonthlyBilling (월 청구액)          // 고객사 × 연월
  - client_id → Client
  - year, month
  - amount

MonthlyDeposit (월 입금액)          // 고객사 × 연월
  - client_id → Client
  - year, month
  - amount

Expense (지출)                      // 고객사 × 연월 × 분류
  - client_id → Client
  - year, month
  - category(법인카드|개인카드|상담료|강사료|홍보비용|기타)
  - amount
  - memo (선택)
```

**설계 노트**
- 과업은 **고객사마다 구성·단가가 다름** → 표준 카탈로그를 강제하지 않고 고객사별 자유 등록. (표준 과업명을 제안 목록으로 보조 가능)
- 실적은 **횟수 입력 → 금액 자동 계산**. `amount`는 조회 성능을 위해 저장하되 항상 `단가 × 횟수`와 일치하도록 관리.
- **부가세는 1단계 범위 제외.** 모든 금액은 부가세 포함 기준.
- 계약금·청구액·입금액이 **없는 경우**를 정상 상태로 취급(0 또는 null 구분 명시).

## 6. 지표 정의

| 지표 | 공식 | 비고 |
|---|---|---|
| **수익률** (대표) | (실적 − 지출) / 실적 | 지출은 고객사 단위 귀속. 고객사별·전사 산출 |
| **실적 달성률** (대표) | 실적 / 계약금 | **계약금이 있는 고객사만** 집계 |
| 청구율 | 청구액 / 실적 | |
| 수금률 | 입금액 / 청구액 | |

- 집계 단위: **고객사별 → 전사(합산) → PM별(본인 담당 합산, 순위 아님)**.
- 기간: 월 기준. 대시보드는 기간 필터(월/상·하반기/연도)로 조회.
- **순위/리더보드는 만들지 않는다** (PM별·고객사별 모두). 개별 수치 조회만 제공.

## 7. 화면 구성

공통 셸: **좌측 세로 사이드바** + 상단 바(기간·고객사 필터, 역할 배지). 역할에 따라 메뉴·데이터가 필터링됨.

1. **메인 대시보드**
   - KPI: 전사 평균 수익률, 실적 달성률, 총 실적, 총 지출, 총 입금
   - 수익 흐름 깔때기: 계약금 → 실적(달성률) → 청구액(청구율) → 입금액(수금률)
   - 월별 추이(수익률·실적 달성률)
   - 지출 구성(6종)
   - PM 로그인 시: 본인 담당 고객사만 집계, 관리자 메뉴 숨김

2. **고객사 목록 / 상세**
   - 목록: 담당 고객사 카드/표 (순위 아님)
   - 상세: 고객사 KPI(수익률·달성률·청구율·수금률), 수익 흐름, 과업별 실적, 월별 실적·청구·입금, 미입금 표시, 내보내기

3. **실적 입력** (PM 핵심 화면)
   - 고객사 + 월 선택 → 과업별 **횟수 입력** → 금액·합계·누계·달성률 자동
   - 과업 추가/수정/삭제(CRUD)

4. **지출 입력**
   - 고객사 + 월 선택 → 6종 분류별 금액 입력(CRUD), 메모

5. **청구·입금 입력**
   - 고객사 + 월 선택 → 청구액·입금액 수동 입력 (2단계에서 파일 업로드 자동화로 대체·보완)

6. **고객사·과업·단가 설정** (관리자/정산담당자)
   - 고객사 등록/수정, 담당 PM 지정, 과업·단가 등록/수정 (2단계에서 PDF 자동생성 대비)

7. **사용자·권한 관리** (관리자 전용)
   - 사용자 승인, 역할 지정, 활성/비활성

## 10. 미해결/추후 확인 항목

- 회계 기준 기간(달력 월 vs 회계연도 경계) — 1단계는 **달력 월 + 연도 필터**로 가정.
- 계약금 없는 고객사의 달성률 표기 방식(현재: "—"으로 제외 표기).
- 횟수 기반이 아닌 과업(예: 정액 홍보비)의 입력 — 1단계는 단가×횟수 모델로 통일하되, 필요 시 "단가=금액, 횟수=1" 방식으로 수용. 실사용 확인 후 조정.
- PM의 실적 입력 후 정산담당자/관리자 **승인 절차** 필요 여부(현재 미포함, 자유 CRUD).
